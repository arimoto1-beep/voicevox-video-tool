# 音声・字幕生成機能 仕様

## 1. 目的

台本ファイルを読み込み、VOICEVOXを使ってセリフごとの音声ファイルを生成する。

また、生成した音声、間、効果音を連結し、動画生成で利用できる全体音声ファイルを作成する。

あわせて、セリフの表示タイミングに対応したSRT字幕ファイルを生成する。

この機能は、将来的な動画生成ツールの前段として利用する。

---

## 2. 入力

### 2.1 台本ファイル

入力は、テキスト形式の台本ファイルとする。

例：

```txt
ずんだもん：印刷開始、うるさくないのだ？

めたん：あーそれ
印刷開始の振動でしょ

(間 0.25)

(SE se\pop.wav)

めたん：原因は、印刷開始時の振動ね
```

台本には、以下の要素を書けるものとする。

- 話者つきセリフ
- 話者省略セリフ
- 読み上げ文と字幕文の分離
- 間
- 効果音
- コメント
- 個別パラメータ

---

## 3. 出力

出力は、以下の3種類とする。

### 3.1 セリフごとのWAVファイル

各セリフについて、VOICEVOXで生成したWAVファイルを出力する。

例：

```txt
wav/001_ずんだもん_印刷開始うるさくないのだ.wav
wav/002_めたん_あーそれ.wav
```

### 3.2 全体音声ファイル

セリフ音声、間、効果音を台本順に連結したWAVファイルを出力する。

例：

```txt
all.wav
```

### 3.3 SRT字幕ファイル

セリフの表示タイミングに対応したSRT字幕ファイルを出力する。

例：

```txt
output.srt
```

SRTには、セリフの字幕だけを出力する。

間や効果音は字幕には出さない。

ただし、間や効果音の時間は、字幕の表示タイミングに反映する。

---

## 4. 台本フォーマット

### 4.1 話者つきセリフ

話者つきセリフは、以下の形式とする。

```txt
ずんだもん：印刷開始、うるさくないのだ？
```

半角コロンも許可する。

```txt
ずんだもん:印刷開始、うるさくないのだ？
```

話者名とセリフを分けて解釈する。

---

### 4.2 話者省略セリフ

話者を省略した行は、直前の話者を引き継ぐ。

```txt
めたん：あーそれ
印刷開始の振動でしょ
```

この場合、2行目も「めたん」のセリフとして扱う。

ただし、直前の話者が存在しない状態で話者省略セリフが出た場合はエラーとする。

---

### 4.3 読み上げ文と字幕文の分離

読み上げ用の文章と字幕用の文章は、`||` で分離できる。

```txt
めたん：あーそれ || あ、それ
```

左側を読み上げ用テキストとする。

右側を字幕用テキストとする。

`||` がない場合は、読み上げ用テキストと字幕用テキストは同一とする。

---

### 4.4 間

間は、以下の形式とする。

```txt
(間 0.25)
```

数値は秒数を表す。

間は全体音声に無音として反映する。

間はSRT字幕には出力しない。

ただし、SRT字幕の時刻計算には反映する。

---

### 4.5 効果音

効果音は、以下の形式とする。

```txt
(SE se\pop.wav)
```

効果音ファイルはWAV形式とする。

相対パスは、台本ファイルの場所を基準に解決する。

効果音は全体音声に反映する。

効果音はSRT字幕には出力しない。

ただし、SRT字幕の時刻計算には反映する。

---

### 4.6 コメント

`#` で始まる行はコメントとして扱い、処理対象外とする。

```txt
# これはコメント
```

コメントは音声にも字幕にも出力しない。

---

### 4.7 空行

空行は無視する。

音声にも字幕にも影響しない。

---

### 4.8 個別パラメータ

セリフまたは効果音には、末尾に個別パラメータを指定できる。

セリフの例：

```txt
めたん：原因はこれよ{speed=1.16}
```

効果音の例：

```txt
(SE se\whoosh.wav){volume=0.35, fade_out=0.03}
```

対応する主なパラメータは以下とする。

セリフ用：

- `speed`
- `pause`

効果音用：

- `volume`
- `fade_in`
- `fade_out`

不明なパラメータが指定された場合はエラーとする。

---

## 5. 処理ルール

処理の大きな流れは、以下とする。

1. 台本ファイルを読み込む
2. 台本をイベントに分解する
3. セリフイベントからVOICEVOX用のリクエストを作る
4. VOICEVOXでセリフごとの音声を生成する
5. 生成したセリフごとのWAVファイルの長さを取得する
6. セリフごとのWAVを保存する
7. 間や効果音を含めて全体音声を連結する
8. セリフ音声、間、効果音の長さを台本順に積み上げ、SRT字幕を生成する

---

## 6. イベントの考え方

台本は、単なる行の集合ではなく、イベント列として扱う。

想定するイベントは以下とする。

- セリフイベント
- 間イベント
- 効果音イベント

コメント行と空行はイベント化しない。

セリフイベントは、以下の情報を持つ。

- 行番号
- 話者
- スタイル
- 読み上げ用テキスト
- 字幕用テキスト
- 個別パラメータ
- 生成されたWAVファイルのパス
- 生成されたWAVファイルの長さ

間イベントは、以下の情報を持つ。

- 行番号
- 秒数

効果音イベントは、以下の情報を持つ。

- 行番号
- ファイルパス
- 個別パラメータ
- 効果音WAVファイルの長さ

将来的には、画像切り替えやタイトル表示もイベントとして追加できるようにする。

---

## 7. VOICEVOX連携

セリフイベントごとに、VOICEVOX APIを呼び出して音声を生成する。

VOICEVOXのエンドポイントは、通常以下を想定する。

```txt
http://127.0.0.1:50021
```

ただし、エンドポイントはオプションで変更できるようにする。

話者名は、VOICEVOX側の話者一覧と照合する。

必要に応じて、別名対応を持つ。

例：

```txt
めたん -> 四国めたん
```

VOICEVOXに接続できない場合は、分かりやすいエラーを出して終了する。

---

## 8. WAV連結

全体音声ファイルは、台本上のイベント順に連結して作成する。

連結対象は以下とする。

- セリフ音声
- 間の無音
- 効果音

WAV形式が一致しない場合はエラーとする。

確認対象は以下とする。

- チャンネル数
- サンプル幅
- サンプリング周波数

効果音の音量やフェード指定がある場合は、連結前に反映する。

---

## 9. SRT生成

SRT字幕は、セリフイベントのみを対象に生成する。

間イベントと効果音イベントは字幕として出力しない。

ただし、時刻計算には含める。

SRTの時刻は、VOICEVOXで生成したセリフごとのWAVファイルの長さをもとに計算する。

台本の文字数や固定秒数だけで字幕時間を決めない。

セリフ音声、間、効果音の長さを台本順に積み上げ、各セリフ字幕の開始時刻と終了時刻を決定する。

つまり、効果音や無音の後に出るセリフは、その分だけ字幕開始時刻が後ろにずれる。

SRTの字幕文には、字幕用テキストを使用する。

読み上げ用テキストとは限らない。

---

## 10. オプション

コマンドラインオプションとして、以下を想定する。

```txt
--script   台本ファイル
--out_dir  個別WAVの出力先
--srt      SRTファイルの出力先
--concat   全体音声ファイルの出力先
--gap      通常のセリフ間の無音秒数
--speed    通常の話速
--pause    通常の句読点間の長さ倍率
--show_speakers  利用可能な話者一覧を表示
```

例：

```txt
python .\make_voicevox_assets.py --script script.txt --out_dir wav --srt output.srt --concat all.wav --gap 0.08 --speed 1.12
```

---

## 11. エラー方針

以下の場合はエラーとする。

- 台本ファイルが存在しない
- 台本ファイルを読み込めない
- 話者省略セリフの前に話者が存在しない
- 指定された話者が見つからない
- VOICEVOXに接続できない
- セリフ音声の生成に失敗した
- 生成したWAVファイルの長さを取得できない
- 効果音ファイルが存在しない
- 効果音ファイルがWAV形式ではない
- 効果音のWAV形式が音声と一致しない
- 効果音WAVファイルの長さを取得できない
- 個別パラメータの形式が不正
- 未対応の個別パラメータが指定された
- 出力先に書き込めない

エラー時は、可能な限り台本の行番号を表示する。

例：

```txt
5行目: 話者 'めたん' が見つかりません
```

エラーは、ただ止まるだけでなく、修正しやすい形で出す。

---

## 12. ログ方針

最低限、以下を表示する。

- 読み込んだイベント数
- 生成したセリフ音声数
- 使用した効果音数
- 出力した個別WAVの件数
- 全体音声ファイルの出力先
- SRTファイルの出力先
- 全体音声の長さ
- SRT字幕件数

大げさなログ機構は最初は不要とする。

ただし、あとから確認できる情報は残す。

---

## 13. テスト観点

主なテスト観点は以下とする。

- 話者つきセリフを解釈できる
- 半角コロンの話者指定を解釈できる
- 話者省略時に直前の話者を引き継げる
- 直前話者がない話者省略セリフはエラーになる
- 読み上げ文と字幕文を分離できる
- `||` がない場合は読み上げ文と字幕文が同一になる
- 間が全体音声に反映される
- 間がSRT時間に反映される
- 間はSRT字幕には出力されない
- 効果音が全体音声に反映される
- 効果音がSRT時間に反映される
- 効果音はSRT字幕には出力されない
- SRTの時刻が、生成済みWAVファイルの長さをもとに計算される
- 台本の文字数や固定秒数だけでSRT時刻を決めていない
- セリフ音声、間、効果音の長さが台本順に積み上げられる
- コメント行が無視される
- 空行が無視される
- 個別パラメータを解釈できる
- 不明な個別パラメータはエラーになる
- VOICEVOX接続失敗時に分かりやすくエラーになる
- セリフ音声の生成失敗時に分かりやすくエラーになる
- 生成済みWAVファイルの長さを取得できない場合に分かりやすくエラーになる
- 効果音ファイルが存在しない場合に分かりやすくエラーになる
- 効果音WAVファイルの長さを取得できない場合に分かりやすくエラーになる
- SRTの字幕時刻が音声の進行と一致する

---
